# AGENTS.md (Project Guidance)

The role of this file is to describe common mistakes and confusion points that agents might encounter as they work in this project. If you ever encounter something in the project that surprises you please alert the developer working with you and indicate this is the case in the AgentMD file to help prevent future agents from having the same issue.

## Project

Native macOS desktop client for LANraragi (manga/doujinshi server), with a deduplication workbench. Apple Silicon-first, macOS 14+.

## Known servers and paths

- Unraid server:
  - Host: `192.168.2.4`
  - User: `root`
  - Auth: SSH keys (no interactive password expected)

## Build

**Requirements:** Xcode 15+, Swift 6, [XcodeGen](https://github.com/yonaskolb/XcodeGen)

```sh
# Regenerate Xcode project after editing project.yml
xcodegen generate

# Open and build/run in Xcode
open LanraragiDesk.xcodeproj

# CLI build
xcodebuild -project LanraragiDesk.xcodeproj -scheme LanraragiDesk -configuration Debug build
```

`.derived/` and build artifacts are runtime output — do not commit them.

**Swift 6 strict mode is active:** all warnings are errors (both Swift and Clang). Fix any warnings introduced by changes before considering work complete.

After code changes, verify with a CLI build and report which files were changed and that the build passed.

When building a version for local testing, replace the installed app in `/Applications/LanraragiDesk.app` with the freshly built app bundle. Do this whenever there is a code change to test.

## Agent gotchas (observed)

- Avoid creating a new `LANraragiClient` for every thumbnail/page byte request. Each client owns its own `URLSession`; doing this in hot paths can spawn excessive concurrent connections and make the macOS UI appear frozen under heavy cover/page loading. Reuse clients per profile inside loader actors.
- Avoid forcing image decode/downsampling through `MainActor` in grid-heavy views (`CoverThumb`, duplicate review page tiles, reader). Decoding many images on the main thread can cause long UI stalls.
- **`Info.plist` is generated by XcodeGen and gets overwritten on every `xcodegen generate`.** Do not edit `Sources/LanraragiDeskApp/Resources/Info.plist` directly — version bumps, ATS keys, and any other plist values must be declared in `project.yml` under `targets.LanraragiDesk.info.properties`. (Observed: a manual 1.1 version bump in the plist was silently reverted to 1.0 by regeneration.)
- `LANraragiClient` instances and their API keys are cached per profile UUID inside `ArchiveLoader`/`ThumbnailLoader`. Profiles keep their UUID across edits, so after changing a profile's base URL or API key you must call `AppModel.invalidateClients(profileID:)` (the profile editor does this) or the app keeps using stale credentials until restart.
- `ArchiveLoader.archiveExists` must only treat HTTP 404/410 as "archive gone". Treating any HTTP error as non-existence caused `loadAndPruneNotDuplicatePairs` to permanently delete the user's "Not a match" decisions on transient 5xx responses.
- Duplicate scans use an incremental fingerprint index: existing fingerprints are reused, and stale ones are pruned by `FingerprintIndexer` after a full enumeration. Do not reintroduce a blanket `resetFingerprintIndex()` before each scan — it forces a re-download of every cover in the library. Full rebuilds are user-triggered via "Rebuild index and scan".
- `not_duplicates.created_at` stores **milliseconds** since the Unix epoch; `profiles`/`index_state` store seconds. Pass milliseconds when supplying explicit timestamps.
- Use the shared cancellable `AsyncLimiter` from `LanraragiKit` for request throttling; do not add per-file private copies (three divergent copies existed once).
