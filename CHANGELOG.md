# Changelog

## Unreleased

### Fixed

- **"Not a match" decisions can no longer be lost to transient server errors.**
  `archiveExists` only treats HTTP 404/410 as "archive gone"; 5xx and other
  errors now abort the Not-a-match refresh instead of pruning saved pairs.
- **Profile edits take effect immediately.** Cached `LANraragiClient`s and API
  keys are invalidated when a profile is saved, so a changed base URL or API
  key no longer requires an app restart.
- **Keychain save failures are surfaced.** The profile editor shows an error
  and blocks the save instead of silently dropping the API key.
- A corrupt `profiles.json` is backed up to `profiles.corrupt.json` and logged
  instead of being silently replaced.
- Deleting a profile also deletes its API key from the Keychain.
- Title-normalization apply results no longer count cancelled items as
  successes.
- Errors shown in the connection status and duplicate-scan UI use short,
  readable messages; full details still go to the activity log.
- Not-a-match management errors are shown in the Not-a-match panel instead of
  overwriting the duplicate-scan status.

### Changed

- **Duplicate scans are incremental.** The cover fingerprint index is reused
  across scans: only new archives are indexed, and fingerprints of archives
  deleted on the server are pruned after each full pass. A new
  "Rebuild index and scan" button (Advanced) forces the old full re-download,
  e.g. after covers changed server-side.
- Request throttling uses a single shared `AsyncLimiter` in LanraragiKit that
  honors task cancellation while queued (previously three private copies that
  ignored cancellation).
- `IndexStore.resetFingerprintIndex` can be scoped to a single profile.
- `LANraragiClient` is now plainly `Sendable` (was `@unchecked`).

### Build / hardening

- Hardened Runtime enabled.
- App Transport Security allows local networking, so plain-HTTP LANraragi
  servers work via local hostnames as well as IP literals.
- App version and ATS settings are declared in `project.yml`
  (`info.properties`); `Info.plist` is generated by XcodeGen and must not be
  edited directly.
- `.gitignore` covers runtime/build output (`.derived/`, `dist/`, `output/`,
  `tmp/`).

## 1.1

- Main page carousels and batch search helpers.
- Improved duplicates review editing, cover actions, and load stability.
